### Identifying Potential Vulnerabilities

- SYSTEM-level access within a domain environment is nearly equivalent to having a domain user account
- Remote Windows exploits: MS08-067, EternalBlue or BlueKeep
- Abusing a service running in the context of the SYSTEM account
- Abusing the service account `SeImpersonate` privileges using Juicy Potato
- Local privesc flaw in Windows OS such as the Windows 10 Task Scheduler 0-day
- Things you can do once you gain SYSTEM-level access on a domain-joined host:
    - Enumerate the domain using built-in tools or offensive tools such as BloodHound and PowerView
    - Perform Kerberoasting / ASREPRoasting attacks within the same domain
    - Run tools such as Inveigh to gather Net-NTLMv2 hashes or perform SMB relay attacks
    - Perform token impersonation to hijack a privileged domain user account
    - Carry out ACL attacks

## Poisoning Attacks

### LLMNR/NBT-NS Poisoning - From Linux

- LLMNR and NBT-NS are used when DNS fails (if LLMNR fails NBT-NS will be used)
- LLMNR port: `5355`, NBT-NS port: `137`
- If SMB signing is disabled we can perform SMB Relay attack
- Tools for LLMNR & NBT-NS poisoning

| **Tool** | **Description** |
| --- | --- |
| Responder | Purpose-built tool to poison LLMNR, NBT-NS, and MDNS, with many different functions |
| Inveigh | Cross-platform MITM platform for spoofing and poisoning attacks |
| Metasploit | Several built-in scanners and spoofing modules for poisoning attacks |

- Directory path where Responder stores captured hashes: `/usr/share/responder/logs`
- Hashcat command for NTLMv2 hashes
```bash
    hashcat -m 5600 hash /usr/share/wordlists/rockyou.txt
```

- Responder usage
```bash
    sudo responder -I eth0
```
> The `-A` flag puts us into analyze mode. Use `-wf` to start the WPAD rogue proxy server and fingerprint remote hosts. Use `-v` for increased verbosity. `-F` and `-P` force NTLM or Basic authentication and force proxy authentication (use sparingly as they may cause a login prompt).


```bash
    # remove new lines and whitespaces from base64 blob
```

## DCSync

- This attack allows an attacker to mimic a Domain Controller by stealing the AD password database using the built-in `Directory Replication Service Remote Protocol`.
- Rights abused: `DS-Replication-Get-Changes-All`
- OPSEC tip: If we have `WriteDacl` on a user with replication rights we could add this privilege to a user we control, execute the DCSync attack and then remove the privileges to cover our tracks.

### Checking a User's Replication Rights

1. Grab SID and group membership
```powershell
    Get-DomainUser -Identity adunn | select samaccountname,objectsid,memberof,useraccountcontrol | fl
```

2. Check Replication Rights
```powershell
    $sid= "S-1-5-21-3842939050-3880317879-2865463114-1164"
    Get-ObjectAcl "DC=inlanefreight,DC=local" -ResolveGUIDs | ? { ($_.ObjectAceType -match 'Replication-Get')} | ?{$_.SecurityIdentifier -match $sid} | select AceQualifier, ObjectDN, ActiveDirectoryRights,SecurityIdentifier,ObjectAceType | fl
```

## Miscellaneous Misconfigurations

### Exchange Related Group Membership

- Members of `Exchange Windows Permissions` can write a DACL on the domain object → DCSync
- Members of `Organization Management` can access mailboxes of all domain users and control `Microsoft Exchange Security Groups`
- Exchange server `PushSubscription` feature can lead to LDAP relay and NTDS dump

### Printer Bug

- Any domain user can trick a server into authenticating to an attacker-controlled machine via the print spooler (runs as SYSTEM)

